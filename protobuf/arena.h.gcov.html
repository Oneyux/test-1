     201             : // Arena allocator. Arena allocation replaces ordinary (heap-based) allocation
     202             : // with new/delete, and improves performance by aggregating allocations into
     203             : // larger blocks and freeing allocations all at once. Protocol messages are
     204             : // allocated on an arena by using Arena::CreateMessage<T>(Arena*), below, and
     205             : // are automatically freed when the arena is destroyed.
     206             : //
     207             : // This is a thread-safe implementation: multiple threads may allocate from the
     208             : // arena concurrently. Destruction is not thread-safe and the destructing
     209             : // thread must synchronize with users of the arena first.
     210             : //
     211             : // An arena provides two allocation interfaces: CreateMessage<T>, which works
     212             : // for arena-enabled proto2 message types as well as other types that satisfy
     213             : // the appropriate protocol (described below), and Create<T>, which works for
     214             : // any arbitrary type T. CreateMessage<T> is better when the type T supports it,
     215             : // because this interface (i) passes the arena pointer to the created object so
     216             : // that its sub-objects and internal allocations can use the arena too, and (ii)
     217             : // elides the object's destructor call when possible. Create<T> does not place
     218             : // any special requirements on the type T, and will invoke the object's
     219             : // destructor when the arena is destroyed.
     220             : //
     221             : // The arena message allocation protocol, required by CreateMessage<T>, is as
     222             : // follows:
     223             : //
     224             : // - The type T must have (at least) two constructors: a constructor with no
     225             : //   arguments, called when a T is allocated on the heap; and a constructor with
     226             : //   a Arena* argument, called when a T is allocated on an arena. If the
     227             : //   second constructor is called with a NULL arena pointer, it must be
     228             : //   equivalent to invoking the first (no-argument) constructor.
     229             : //
     230             : // - The type T must have a particular type trait: a nested type
     231             : //   |InternalArenaConstructable_|. This is usually a typedef to |void|. If no
     232             : //   such type trait exists, then the instantiation CreateMessage<T> will fail
     233             : //   to compile.
     234             : //
     235             : // - The type T *may* have the type trait |DestructorSkippable_|. If this type
     236             : //   trait is present in the type, then its destructor will not be called if and
     237             : //   only if it was passed a non-NULL arena pointer. If this type trait is not
     238             : //   present on the type, then its destructor is always called when the
     239             : //   containing arena is destroyed.
     240             : //
     241             : // - One- and two-user-argument forms of CreateMessage<T>() also exist that
     242             : //   forward these constructor arguments to T's constructor: for example,
     243             : //   CreateMessage<T>(Arena*, arg1, arg2) forwards to a constructor T(Arena*,
